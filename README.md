## Git 컨벤션 개요
Conflict를 방지하고, 효과적이고 명확한 협업을 진행하고자 Git-Flow 브랜치 전략을 도입하여 계층별 브랜치를 관리했습니다.

### Git Flow
```
master   
└ develop  
  ├ front - feature/FE-기능...  
  └ back - feature/BE-기능...
```
  
master : 운영 서버로 배포하기 위한 브랜치
develop : 다음 출시 기능을 개발하는 브랜치
front : 프론트엔드 개발하는 브랜치
back : 백엔드를 개발하는 브랜치
feature : 세부 기능을 개발하는 브랜치

feature- 이후에 어떤 기능을 개발하는지를 붙여주었습니다.
